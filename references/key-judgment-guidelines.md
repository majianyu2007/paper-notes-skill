# Key Judgment Guidelines

Use this file when the note should read like a research-oriented technical analysis rather than a plain summary.

## Goal

Write objective but high-density analytical judgments.

The point is not to sound emotional or subjective. The point is to identify the paper's real argumentative spine.

## What a key judgment is

A key judgment is a compressed analytical claim that answers one of these questions:

- what problem the paper is really solving
- what the method is really doing
- what the crucial operator or mechanism is
- what the experiments actually prove

It should be supportable by the paper's figures, formulas, tables, or code.

## Recommended layers

Use one or more of these layers when useful:

### 1. 问题定义层判断

Answer:

- what contradiction or bottleneck the paper is actually targeting
- why prior work was insufficient

Typical phrasing:

- 这篇工作的真正问题不在于……而在于……
- 更准确地说，它处理的是……
- 论文真正瞄准的矛盾是……

### 2. 方法层判断

Answer:

- what the method's main structural move is
- how the modules divide labor

Typical phrasing:

- 方法真正关键不在于模块数量，而在于……
- 从整体设计看，作者采用的是……
- 这个框架最核心的结构动作是……

### 3. 算子 / 模块层判断

Answer:

- what a specific block is really doing
- what it is not doing

Typical phrasing:

- 它不是简单拼接，而是……
- 这里最关键的不是卷积本身，而是……
- 这个模块更接近……而不是……

### 4. 实证层判断

Answer:

- what the evidence chain really supports
- whether the paper proves only effectiveness, or also mechanism / transferability

Typical phrasing:

- 这组结果说明……
- 这些实验支持的不是单点涨分，而是……
- 证据并没有停留在……而是进一步表明……

## Writing constraints

- keep tone objective
- every strong judgment should be evidence-backed
- do not force all four layers into every paper
- if the paper is simple, one or two layers may be enough

## Good behavior

Prefer:

- short judgment sentence first
- then explanation
- then figure / formula / table / code support

This sequence often works well:

1. key judgment
2. mechanism explanation
3. evidence support

## Avoid

- empty evaluative praise
- fake certainty without evidence
- vague statements like “方法很有效” without analytical content
- repeating the abstract and calling it judgment

## Mini checklist

Before keeping a judgment paragraph, verify:

- it answers a real analytical question
- it is not just abstract paraphrase
- it can be traced back to evidence
- it sounds like analysis, not marketing
